Документація проєкту: ExamSimulator
1. Призначення програми
ExamSimulator - це консольний застосунок, призначений для створення, редагування та проходження навчальних тестів. Програма створена для автоматизації процесу самоперевірки знань користувачів, дозволяючи їм тренуватися на заздалегідь підготовлених наборах запитань, отримувати миттєвий зворотний зв'язок та відстежувати свою успішність з різних дисциплін у часі.

2. Короткий опис предметної області
Предметна область охоплює сферу електронного навчання та тестування.
Основними сутностями є:
- Тема (Topic) - глобальна дисципліна (наприклад, "Основи програмування").
- Тест (Test) - специфічний розділ у межах теми (наприклад, "ООП").
- Запитання (Question) - одиниця перевірки знань, яка має рівень складності (Easy, Medium, Hard). Запитання можуть бути різних типів: з однією правильною відповіддю, з декількома (де передбачені часткові бали та штрафи за помилки) та відкриті запитання (введення тексту вручну).
- Статистика (Statistics) - результати спроб користувача, які дозволяють аналізувати відсоток засвоєння матеріалу.

3. Основні можливості системи
Система забезпечує виконання наступних функцій:
- Керування контентом: Створення та видалення тем, додавання нових тестів до тем, конструювання запитань різних типів прямо через консольний інтерфейс.
- Генерація тренувальних сесій: Формування тесту із заданою кількістю запитань та перемішуванням варіантів відповідей для уникнення механічного запам'ятовування позицій.
- Система оцінювання: Гнучке нарахування балів залежно від типу запитання (з підтримкою штрафів за вибір неправильних опцій у питаннях з множинним вибором).
- Збір статистики: Фіксація результатів кожної сесії з відображенням загальної історії та розрахунком середньої успішності по кожній темі.
- Динамічні налаштування: Можливість керувати параметрами системи (кількість запитань у сесії, показ еталонних відповідей) без перезапуску програми.

4. Інструкція зі збирання та запуску
Системні вимоги:

Встановлений .NET 10.0 SDK.

Запуск через Visual Studio / Rider:

Відкрити файл рішення ExamSimulator-kpi.slnx.
Переконатися, що проєкт ExamSimulator встановлено як стартап-проєкт (Startup Project).
Натиснути кнопку Run (або F5).

Запуск через командний рядок (Terminal/Console):

Відкрити термінал у кореневій папці проєкту (там, де лежить файл рішення).
Виконати команду для запуску:
                                dotnet run --project ExamSimulator
Для запуску тестів виконати:
                                dotnet test ExamSimulator.Tests

5. Інструкція з використання
Після запуску програми користувач потрапляє в Головне меню. Навігація здійснюється шляхом введення відповідної цифри та натискання Enter. Цифра 0 завжди використовується для повернення до попереднього меню або скасування поточної дії.

Тренування: Оберіть пункт 1. Далі оберіть номер теми, потім номер тесту. Читайте запитання і вводьте відповідь. Для запитань з множинним вибором вводьте номери через кому (наприклад, 1,3).

Редагування бази: Оберіть пункт 2. Ви можете додати нову тему (пункт 1), тест (пункт 4) або запитання (пункт 5). Слідуйте підказкам на екрані.

Налаштування: Оберіть пункт 3, щоб змінити кількість запитань, які видаються за одну сесію, або вимкнути підказки після неправильних відповідей.

Статистика: Оберіть пункт 4 для перегляду таблиці ваших минулих результатів.

6. Формат і спосіб збереження даних
Формат: Дані зберігаються у текстових файлах у форматі JSON. Це забезпечує легку читабельність файлів людиною та зручність резервного копіювання. Під час збереження використовується налаштування UnicodeRanges.Cyrillic для коректного відображення української мови.

Спосіб збереження: Використовується патерн Repository у поєднанні з DataStorage. Під час запуску програми дані завантажуються з файлів в оперативну пам'ять (у списки List<T>). Усі зміни відбуваються в пам'яті, а після кожної успішної операції створення/зміни/видалення викликається метод SaveChanges(), який повністю перезаписує відповідний JSON-файл актуальним станом.

7. Перелік і призначення підготовлених файлів із прикладами даних
У папці data/ містяться три попередньо заповнені файли для перевірки функціональності:
- topics.json - База знань. Містить 3 повноцінні теми ("Основи програмування", "Математичний аналіз", "Алгоритми та структури даних"). Кожна тема містить по 3 тести, в кожному з яких знаходиться по 10 запитань різної складності та різних типів (загалом 90 запитань).
- statistics.json - Історія користувача. Містить зразок збережених попередніх сесій проходження тестів (з різними балами). Дозволяє одразу перевірити працездатність модуля статистики та розрахунку середнього бала без необхідності вручну проходити тести.
- appsettings.json - Файл налаштувань. Містить базову конфігурацію системи за замовчуванням (наприклад, видавати по 5 питань).

8. Основні модулі, компоненти та їхня відповідальність
Система побудована за багатошаровою архітектурою:

Модуль Domain (Сутності): Містить моделі даних (Topic, Test, Question, UserStatistics). Реалізує ключову бізнес-логіку на рівні моделей - поліморфний метод CalculateScore, який по-різному нараховує бали залежно від типу запитання.

Модуль Data (Зберігання): 
                                - GenericRepository<T> - відповідає за CRUD-операції в оперативній пам'яті.
                                - JsonDataStorage<T> - відповідає безпосередньо за серіалізацію/десеріалізацію та читання/запис файлів на диск.

Модуль Services (Бізнес-сервіси):
                                - TrainingService - відповідає за алгоритми генерації сесії (вибір запитань за лімітом) та перемішування варіантів відповідей.
                                - ContentManagerService - керує зв'язками між репозиторіями при створенні бази запитань.
                                - StatisticsService - розраховує аналітику та агрегує дані користувача.

Модуль UI (Інтерфейс): Контролери (TrainingController, EditorController, SettingsController), які відповідають виключно за взаємодію з користувачем через Console.ReadLine / Console.WriteLine та передачу цих команд у сервіси.

9. Конфігурація проєкту
Конфігурація системи реалізована через об'єкт моделі AppConfig, який керується сервісом ConfigManager. Усі налаштування зберігаються у файлі appsettings.json.
Система підтримує такі параметри конфігурації:
- QuestionsPerSession (за замовчуванням: 5) - Кількість запитань, яка випадковим чином видається користувачу при запуску одного тесту (запобігає видачі всіх 100 запитань одночасно).
- ShowCorrectAnswersAfterQuestion (за замовчуванням: true) - Булевий прапорець, який керує режимом навчання: якщо увімкнено, після неправильної відповіді система покаже еталонний варіант.
- PassingScorePercentage (за замовчуванням: 60.0) - Відсоток правильних відповідей, необхідний для того, щоб тест вважався успішно складеним.
